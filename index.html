01

What's inside

Most “open” models ship weights but hide the machinery. Here every component is in the repo, in a few hundred readable lines each.

Tokenizer

Byte-level BPE

Trained from scratch. Round-trips any Unicode with no <unk> — trains on a frequency-weighted multiset so it scales.

Attention

GQA + RoPE + QK-norm

Grouped-query attention shrinks the KV cache; rotary embeddings and QK-normalization keep training stable at higher learning rates.

Norm / FFN

RMSNorm · SwiGLU

Pre-norm residual blocks with gated feed-forward networks — the modern default that every frontier model converged on.

Sparsity

Mixture-of-Experts

Fine-grained top-k routing, shared experts, and a Switch-style load-balancing loss. Flip one field to go dense → sparse.

Training

Real training loop

Cosine schedule with warmup, gradient accumulation and clipping, eval/perplexity, rotating checkpoints — no framework magic.

Inference

KV-cached decoding

O(1)-per-token generation with temperature, top-k, top-p, and repetition penalty. Proven identical to a full forward pass by test.

02

Dense or sparse — one config field

Mixture-of-Experts is the mechanism behind “huge total, small active” frontier models. In MuseLM it’s a single switch.

Dense
n_experts = 0
≈ 5.0M params · every parameter active
  • Standard SwiGLU feed-forward
  • Simple, fast, great baseline
Mixture-of-Experts
n_experts = 8 · top-k = 2
≈ 16.3M total · only 2 experts routed per token
  • More capacity at ~constant compute
  • Shared experts + load-balancing loss

Quickstart

Clone, install, and go from a raw text file to sampled text — tokenizer, data, model, generation.

# install pip install -e ".[dev]" # train a byte-BPE tokenizer, tokenize, and train (auto CUDA/MPS/CPU) muselm tokenizer-train --input corpus.txt --output tok.json --vocab-size 2048 muselm prepare --input corpus.txt --tokenizer tok.json --output data/ muselm train --config configs/tiny.json # or skip training — use the checkpoint that ships in the repo muselm generate --checkpoint checkpoints/release/best.pt \ --tokenizer checkpoints/release/tokenizer.json \ --prompt "To be, or not to be"
